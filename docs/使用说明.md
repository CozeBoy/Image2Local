# Image2Local 使用说明

**语言：** [English](usage.md) | [中文](使用说明.md)

## 概述

Image2Local 是一款 Obsidian 社区插件，用于将笔记中的**在线图片**和 **Base64 编码图片**转存到本地文件系统，并自动更新为 Markdown 图片引用语法。

## 需求对照

| 需求 | 实现方式 |
|------|----------|
| 自动保存在线图片到本地 | 开启自动转存后，粘贴或保存笔记时自动处理 |
| 支持链接和 Base64 | 识别 `![alt](https://...)` 和 `![alt](data:image/...;base64,...)` |
| Markdown 语法引用 | 转存后替换为 `![alt](相对路径)` |
| 可配置存储目录 | 设置页配置路径模式与目录名，默认 `image_files` |
| 绝对/相对路径 | 四种模式：相对文档、相对仓库根、仓库内绝对路径、系统绝对路径 |
| 中英文切换 | 设置 → 界面语言 |
| 右键转存 | 图片右键（含 Base64）及编辑器右键菜单 |
| 批量转存 | 识别 Markdown 图片、HTML img、裸 URL、Base64 |
| 内置 10 个 User-Agent | 设置页选择固定 UA 或开启随机轮换 |

## 设置项说明

### 开启自动转存

启用后，当你：

- 在编辑器中**粘贴**含图片的内容
- **保存**当前正在编辑的 Markdown 笔记

插件会自动扫描笔记中的在线图片和 Base64 数据，下载/解码后保存到本地，并替换原文中的引用。

> 自动转存仅处理**当前正在编辑**的笔记，不会影响仓库中其他文件。

### 存储路径模式

#### 1. 相对文档路径（默认）

图片保存在当前笔记所在目录下的子文件夹中。

```
我的笔记/
├── 某篇文章.md
└── image_files/
    └── image_1234567890_abc123.png
```

笔记中的引用：`![示例](./image_files/image_1234567890_abc123.png)`

#### 2. 相对仓库根路径

图片保存在仓库根目录下的指定文件夹（默认为 `image_files`）。

```
仓库根/
├── image_files/
│   └── image_1234567890_abc123.png
└── 文件夹/
    └── 某篇文章.md
```

笔记中的引用会根据笔记位置自动计算相对路径。

#### 3. 绝对路径（仓库内）

图片保存在仓库内你指定的绝对路径，例如 `attachments/images`。

```
仓库根/
├── attachments/
│   └── images/
│       └── image_1234567890_abc123.png
└── 笔记.md
```

#### 4. 系统绝对路径（仓库外，仅桌面端）

图片保存在操作系统中的任意目录，例如 `/Users/你/Pictures/image_files`。

```
/Users/你/Pictures/image_files/
└── image_1234567890_abc123.png
```

笔记中的引用会使用 `file://` 格式，以便 Obsidian 预览仓库外的图片。

### User-Agent 设置

部分网站会拦截没有浏览器标识的下载请求。插件内置 10 个常见 User-Agent：

1. Chrome (Windows)
2. Chrome (macOS)
3. Chrome (Linux)
4. Firefox (Windows)
5. Firefox (macOS)
6. Safari (macOS)
7. Edge (Windows)
8. Safari (iPhone)
9. Chrome (Android)
10. Safari (iPad)

- **随机轮换**：每次下载随机选择一个（推荐）
- **固定选择**：关闭随机轮换后，可指定使用某一个

## 操作方式

### 方式一：自动转存（推荐）

1. 确保已开启「自动转存」
2. 在笔记中粘贴图片或图片链接
3. 等待自动处理，或手动保存笔记（Ctrl/Cmd + S）

### 方式二：右键菜单

**Live Preview / 阅读模式：**

1. 在在线图片或 Base64 图片上右键
2. 选择「转存图片到本地」

**编辑器 / 源码模式：**

1. 将光标放在图片链接或 Base64 数据上
2. 右键 →「转存图片到本地」

**批量转存：** 右键 →「转存笔记中所有图片 (N)」或「批量转存笔记内全部图片」

### 方式三：命令面板

1. 按 `Ctrl/Cmd + P` 打开命令面板
2. 搜索「批量转存当前笔记中的图片」
3. 回车执行

## 支持的图片源

### 在线链接

```markdown
![示例](https://example.com/image.png)
![示例](https://example.com/photo.jpg?size=large)
```

### Base64 数据

```markdown
![示例](data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...)
```

### HTML 图片标签

```html
<img src="https://example.com/image.png" alt="示例">
```

### 裸图片链接

```markdown
https://example.com/image.png
```

## 网络与隐私

- 插件仅下载您笔记中出现的图片 URL
- 不收集遥测或分析数据
- User-Agent 仅用于提高图片下载成功率

## 平台说明

- **桌面端**：完整功能，含系统绝对路径
- **移动端**：不支持（插件使用 Node.js 文件系统 API）

## 常见问题

### 图片下载失败？

- 检查网络连接
- 尝试开启「随机轮换 User-Agent」
- 部分网站有防盗链限制，可能无法下载

### 转存后路径不对？

- 检查「存储路径模式」是否符合预期
- 相对路径是相对于**当前笔记文件**计算的

### 自动转存没有触发？

- 确认已开启「自动转存」
- 确认正在编辑的是 Markdown 笔记
- 粘贴后稍等片刻，或手动保存笔记

## 技术说明

- 使用 Obsidian 的 `requestUrl` API 下载远程图片
- 系统绝对路径使用 Node.js `fs` 模块写入文件
- 图片文件名根据 URL 或时间戳自动生成，避免重名冲突
- 转存过程有防重入保护，避免循环触发
